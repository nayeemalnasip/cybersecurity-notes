# What is Networking? - Project Summary

## Project Overview

This project focused on understanding the fundamental concepts of computer networking. Throughout the room, I explored how networks are formed, how the Internet operates, how devices are identified on a network, and how connectivity can be tested using ICMP and Ping.

---

## Task Completion Process

### Understanding Network Fundamentals

The first step was understanding what a network is and why networking is important in modern computing. I learned that a network is simply a collection of connected devices that communicate and exchange information with one another.

By studying real-world and computing-based examples, I developed a foundational understanding of how networks support communication between devices and services.

**Outcome:**

- Identified the concept of a network.
- Understood how connected devices form a communication environment.

---

### Exploring the Internet

Next, I examined how the Internet is structured and how it differs from a standard network.

I learned that the Internet is a global collection of interconnected networks. During this task, I explored the historical development of the Internet and discovered the contribution of Tim Berners-Lee in creating the World Wide Web (WWW).

**Outcome:**

- Understood the relationship between networks and the Internet.
- Learned the role of the World Wide Web in information sharing.
- Explored the evolution of modern Internet communication.

---

### Identifying Devices on a Network

The third task focused on how devices are uniquely identified within a network.

I explored:

- Internet Protocol (IP) addressing
- IPv4 structure
- Public and Private IP addresses
- Media Access Control (MAC) addresses

I learned that IPv4 addresses are divided into four octets and that MAC addresses provide a unique hardware identifier for network interfaces.

As part of the practical exercise, I performed a MAC spoofing activity to understand how network devices can impersonate trusted systems when security controls rely solely on MAC-based identification.

**Practical Activity Completed:**

```text
THM{YOU_GOT_ON_TRYHACKME}
```

**Outcome:**

- Understood logical and physical device identification.
- Learned the structure of IPv4 addressing.
- Explored MAC spoofing concepts and their security implications.

---

### Network Diagnostics with Ping

In the final task, I explored ICMP and the Ping utility.

I used Ping to understand how devices verify connectivity across a network using ICMP Echo Requests and Echo Replies. This demonstrated how administrators can test network reachability and measure response times between devices.

**Command Used**

```bash
ping 10.10.10.10
```

**Practical Activity Completed:**

```text

```

**Outcome:**

- Learned the role of ICMP in network communication.
- Tested device connectivity using Ping.
- Understood basic network troubleshooting techniques.
