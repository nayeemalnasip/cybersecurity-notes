# Intro to LAN - Project Summary

## Project Overview

This project focused on understanding the core concepts of Local Area Networks (LANs) and how devices communicate within a local network environment. Throughout the room, I explored different network topologies, subnetting, address resolution using ARP, and automatic IP address assignment through DHCP.

The room provided both theoretical knowledge and practical exposure to fundamental networking concepts that are essential for cybersecurity and network administration.

---

## Task Completion Process

### Exploring LAN Topologies

The first task focused on understanding how devices are connected within a Local Area Network (LAN). I studied three common network topologies: Star, Bus, and Ring.

I learned how each topology handles communication between devices and analyzed their advantages, disadvantages, scalability, reliability, and fault tolerance.

Additionally, I explored the roles of switches and routers within a network. I learned that switches are responsible for forwarding traffic to the correct destination within a LAN, while routers perform routing between different networks.

As part of the practical exercise, I analyzed vulnerable network designs and identified weaknesses within different topologies.

**Outcome:**

* Understood the concept of LAN topologies.
* Compared Star, Bus, and Ring topologies.
* Learned the functions of switches and routers.
* Identified common network design weaknesses.

---

### Understanding Subnetting

The second task introduced subnetting and its role in dividing large networks into smaller and more manageable segments.

I learned how subnet masks are used to separate network and host portions of an IP address. I also explored the three important address types used within a subnet:

* Network Address
* Host Address
* Default Gateway

Through this task, I understood how organizations use subnetting to improve network organization, security, and efficiency.

**Outcome:**

* Understood the purpose of subnetting.
* Learned the structure of subnet masks.
* Identified Network, Host, and Gateway addresses.
* Explored how subnetting improves network management.

---

### Learning Address Resolution Protocol (ARP)

The third task focused on Address Resolution Protocol (ARP), which enables devices to associate IP addresses with MAC addresses.

I explored how devices maintain ARP caches and how ARP Requests and ARP Replies are used to discover physical device addresses on a local network.

By understanding the relationship between logical and physical addressing, I gained insight into how devices identify and communicate with one another within a LAN.

**Outcome:**

* Learned the purpose of ARP.
* Understood the relationship between IP and MAC addresses.
* Explored ARP Request and ARP Reply communication.
* Learned how ARP caches improve network efficiency.

---

### Understanding Dynamic Host Configuration Protocol (DHCP)

The final task introduced DHCP and how devices automatically obtain IP addresses when joining a network.

I studied the complete DHCP communication process, including:

* DHCP Discover
* DHCP Offer
* DHCP Request
* DHCP ACK

This process demonstrated how network devices receive valid IP addresses and network configurations without requiring manual setup.

**Outcome:**

* Understood automatic IP address assignment.
* Learned the DHCP DORA process.
* Explored how DHCP simplifies network administration.
* Understood the role of DHCP servers in modern networks.
