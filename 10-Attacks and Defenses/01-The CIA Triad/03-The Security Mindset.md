# Cyber Security Fundamentals — Study Note

## The Security Mindset

### Overview

Cyber security is not only about using security tools or defending against attacks. It also involves adopting a security mindset that helps professionals analyze incidents, understand their impact, and make informed decisions.

One of the most important frameworks used by security professionals is the **CIA Triad**:

```text
Confidentiality
Integrity
Availability
```

When a security incident occurs, cyber security professionals evaluate which component of the CIA Triad has been affected. This approach helps identify risks, assess damage, prioritize responses, and implement appropriate security controls.

---

## Main Concept

### What is the Security Mindset?

The security mindset is the ability to think critically about how information, systems, and services can be protected or compromised.

Rather than simply reacting to incidents, security professionals ask questions such as:

```text
Was sensitive information exposed?
Was data modified without authorization?
Were services unavailable when needed?
```

These questions help determine which pillar of the CIA Triad has been affected.

---

## How It Works

### Step 1: Identify the Incident

A security event or abnormal activity occurs.

Examples:

```text
Data Leak
Website Outage
Unauthorized File Modification
```

---

### Step 2: Analyze the Impact

Determine what happened and what assets were affected.

Questions:

```text
Was information disclosed?
Was information altered?
Was access disrupted?
```

---

### Step 3: Map to the CIA Triad

Associate the incident with the affected security principle.

| Incident Type      | CIA Component   |
| ------------------ | --------------- |
| Data Exposure      | Confidentiality |
| Data Tampering     | Integrity       |
| Service Disruption | Availability    |

---

### Step 4: Determine Response

Security teams select controls and remediation actions based on the affected pillar.

Examples:

```text
Confidentiality → Strengthen Access Controls
Integrity → Verify Data Accuracy
Availability → Restore Services and Redundancy
```

---

## Important Components

### Confidentiality Assessment

Questions Security Professionals Ask:

```text
Was sensitive data viewed by unauthorized users?
Were passwords exposed?
Was private information leaked?
```

Examples:

* Stolen login credentials
* Leaked customer records
* Unauthorized document access

Affected Principle:

```text
Confidentiality
```

---

### Integrity Assessment

Questions Security Professionals Ask:

```text
Was data altered?
Was information manipulated?
Can the data still be trusted?
```

Examples:

* Modified bank transactions
* Altered grades
* Tampered business records

Affected Principle:

```text
Integrity
```

---

### Availability Assessment

Questions Security Professionals Ask:

```text
Can users access the service?
Is the system operational?
Has business activity been interrupted?
```

Examples:

* Website outages
* Server failures
* Denial-of-Service attacks

Affected Principle:

```text
Availability
```

---

### Incident Analysis Process

Security professionals typically follow this workflow:

```text
Security Incident
       ↓
Determine Impact
       ↓
Identify CIA Component
       ↓
Assess Severity
       ↓
Implement Response
       ↓
Restore Security
```

---

## Advantages / Benefits

* Improves incident response effectiveness.
* Helps identify security weaknesses.
* Supports risk assessment.
* Provides a structured analysis method.
* Improves security decision-making.
* Enhances understanding of attack impacts.
* Supports business continuity planning.

---

## Key Characteristics

* Security incidents are analyzed using the CIA Triad.
* Every incident affects one or more CIA components.
* Security professionals focus on impact assessment.
* The CIA Triad serves as a decision-making framework.
* Understanding the affected pillar helps prioritize response actions.
* Multiple CIA principles may be impacted by a single attack.
