# Defensive Security Fundamentals

## Introduction to Defensive Security

### Overview

Defensive Security is the branch of cybersecurity focused on protecting systems, networks, applications, and data from cyber threats. Unlike offensive security, which identifies and tests weaknesses, defensive security aims to prevent attacks, detect malicious activity, minimize damage, and ensure the continued availability of critical services.

Defensive security supports the principles of the **CIA Triad (Confidentiality, Integrity, and Availability)** by implementing security controls, monitoring systems, and responding to incidents when they occur.

---

## Main Concept

### What is Defensive Security?

Defensive Security is the practice of protecting information systems by implementing safeguards, monitoring for threats, detecting suspicious activity, and responding to security incidents.

The primary goal is to reduce risk and maintain secure, reliable operations.

### Core Objective

```text
Protect
↓
Detect
↓
Respond
↓
Recover
```

Defenders continuously work to secure systems before, during, and after cyber attacks.

### Step 1: Identify Critical Assets

Determine what needs protection.

Examples:

```text

User Accounts

Cloud Resources
```

---

### Step 2: Gain Visibility

Monitor systems and network activity to understand normal operations.

Examples:

```text

Network Traffic
User Activity
Security Alerts
```

---

### Step 3: Implement Security Controls

Deploy protective measures to reduce risk.

Examples:

```text
Firewalls
Antivirus
Access Controls
Encryption
```

---

### Step 4: Detect Threats

Monitor for suspicious or malicious activity.

Examples:

```text
Unauthorized Login Attempts
Malware Detection
Network Scanning
Data Exfiltration
```

---

### Step 5: Respond to Incidents

Take action when threats are identified.

Examples:

```text
Block Malicious Traffic
Disable Compromised Accounts
Isolate Infected Systems
```

---

### Step 6: Recover and Improve

Restore affected services and strengthen defenses.

Examples:

```text
System Restoration
Patch Vulnerabilities
Security Improvements
```

## Important Components

### Blue Team

The Blue Team is responsible for defending systems and responding to security threats.

#### Responsibilities

* Monitor systems
* Detect attacks
* Investigate incidents
* Improve security controls

---

### Visibility

Visibility refers to the ability to observe and understand activity occurring within an environment.

#### Why It Matters

Without visibility:

```text

↓

↓
```

* Security Alerts

* User Activity

## Relationship Between Offensive and Defensive Security

### Offensive Security

Focuses on:

```text
Finding Weaknesses
Testing Defenses
Simulating Attacks
```

---

### Defensive Security

Focuses on:

```text
Preventing Attacks
Detecting Threats
Responding to Incidents
```

---

### Working Together

```text
Offensive Security
↓
Identifies Weaknesses
↓
Defensive Security
↓
Implements Protections
```

Both disciplines are essential for effective cybersecurity.
