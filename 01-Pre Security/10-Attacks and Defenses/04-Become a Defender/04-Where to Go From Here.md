# Defensive Security Fundamentals

## Defensive Security Career Path and Future Learning

### Overview

Defensive Security focuses on protecting an organization's systems, users, applications, and data from cyber threats. Throughout this room, you learned how defenders identify critical assets, gain visibility into their environments, anticipate attacks, and implement layered security controls to reduce risk.

Effective defenders understand both how systems operate and how attackers think, allowing them to detect, prevent, and respond to threats before significant damage occurs.

---

## Main Concept

### What is Defensive Security?

Defensive Security is the practice of protecting digital assets through monitoring, prevention, detection, mitigation, and incident response activities.

The ultimate goal is to maintain:

* Confidentiality
* Integrity
* Availability

while ensuring business operations continue securely.

### Core Security Lifecycle

```text
Identify Assets
↓
Monitor Systems
↓
Detect Threats
↓
Respond to Incidents
↓
Recover Operations
↓
Improve Security
```

---

## How It Works

### Asset Identification

Defenders first identify critical infrastructure.

Examples:

```text
Servers
Workstations
Web Applications
Databases
Cloud Resources
User Accounts
```

---

### Visibility and Monitoring

Continuous monitoring provides awareness of system activity.

Sources include:

```text
Security Logs
Network Traffic
Authentication Events
Security Alerts
```

---

### Threat Detection

Security tools identify abnormal or malicious behavior.

Examples:

```text
Failed Login Attempts
Malware Activity
Unauthorized Access
Suspicious Network Connections
```

---

### Incident Response

When threats are detected, defenders investigate and contain the issue.

Examples:

```text
Disable Accounts
Block Malicious IPs
Isolate Systems
Remove Malware
```

---

### Continuous Improvement

After incidents, organizations improve defenses through:

```text
Patch Management
Security Awareness
Monitoring Enhancements
Policy Updates
```

## Important Components

### Blue Team

The Blue Team is responsible for protecting organizational assets and responding to security incidents.

#### Responsibilities

* Monitoring systems
* Investigating alerts
* Managing security tools
* Incident response
* Security improvements

---

### Client Infrastructure

Infrastructure includes all systems and technologies owned or managed by an organization.

Examples:

| Infrastructure Component | Purpose             |
| ------------------------ | ------------------- |
| Employee Devices         | User Productivity   |
| Web Servers              | Website Hosting     |
| Mail Servers             | Email Communication |
| Databases                | Data Storage        |
| Firewalls                | Traffic Control     |
| Cloud Services           | Business Operations |

---

### Visibility

Visibility refers to the ability to observe activity throughout the environment.

#### Examples

```text
System Logs
Network Monitoring
Endpoint Monitoring
Security Dashboards
```

Without visibility, threats may go undetected.

---

### Threats

Threats are potential dangers capable of compromising systems or data.

Examples:

```text
Hackers
Malware
Phishing
Ransomware
Insider Threats
```

---

### Prevention

Security controls designed to stop attacks before they occur.

Examples:

| Control   | Purpose                 |
| --------- | ----------------------- |
| Firewall  | Network Protection      |
| Antivirus | Malware Prevention      |
| MFA       | Account Security        |
| Patching  | Vulnerability Reduction |

---

### Detection

The process of identifying suspicious activity.

Examples:

```text
SIEM Alerts
Log Analysis
Endpoint Monitoring
Threat Detection Systems
```

---

### Mitigation

Actions taken to limit damage from an active threat.

Examples:

```text
Block Traffic
Disable Accounts
Quarantine Systems
Remove Malware
```

---

### Risk

Risk represents the likelihood and impact of a successful attack.

Formula:

```text
Threat + Vulnerability = Risk
```

Defenders work to reduce risk through security controls and monitoring.
