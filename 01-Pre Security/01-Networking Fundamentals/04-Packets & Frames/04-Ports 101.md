# Ports 101

## Overview

Ports are logical communication endpoints used by networking devices to send and receive data. Every application or service that communicates over a network uses a specific port number. Ports help operating systems identify which application should handle incoming or outgoing network traffic.

---

## What is a Port?

A port is a numerical communication channel that allows data to be exchanged between devices and applications.

Think of a port like a docking station at a harbor:

* Ships dock at specific ports.
* Only compatible ships can use specific docking areas.
* Ports control where traffic enters and leaves.

Similarly, network ports control how applications communicate across a network.

---

## Port Number Range

Network ports use numerical values ranging from:

```text
0 - 65535
```

Each port number can be assigned to a specific application or service.

---

## Why Are Ports Important?

Many applications run simultaneously on a computer.

Examples:

* Web Browsers
* Email Clients
* File Sharing Services
* Remote Access Tools

Ports ensure that incoming data is delivered to the correct application.

Without ports, the operating system would not know which application should receive the network traffic.

---

## Common Ports

Ports between:

```text
0 - 1024
```

are known as **Common Ports** or **Well-Known Ports**.

These ports are reserved for widely used networking protocols.

---

## Common Protocols and Their Ports

| Protocol | Port | Purpose                |
| -------- | ---- | ---------------------- |
| FTP      | 21   | File Transfer          |
| SSH      | 22   | Secure Remote Access   |
| HTTP     | 80   | Web Browsing           |
| HTTPS    | 443  | Secure Web Browsing    |
| SMB      | 445  | File & Printer Sharing |
| RDP      | 3389 | Remote Desktop Access  |

---

## FTP (File Transfer Protocol)

**Port:** 21

Purpose:

* Upload files
* Download files
* Client-server file sharing

---

## SSH (Secure Shell)

**Port:** 22

Purpose:

* Secure remote administration
* Command-line access to systems

---

## HTTP (HyperText Transfer Protocol)

**Port:** 80

Purpose:

* Transfers webpages
* Powers the World Wide Web (WWW)

---

## HTTPS (HyperText Transfer Protocol Secure)

**Port:** 443

Purpose:

* Secure web communication
* Encrypts transmitted data

---

## SMB (Server Message Block)

**Port:** 445

Purpose:

* File sharing
* Printer sharing
* Resource sharing on networks

---

## RDP (Remote Desktop Protocol)

**Port:** 3389

Purpose:

* Remote graphical desktop access
* Remote system administration

---

## Standard Ports vs Custom Ports

Protocols follow standard ports by convention.

Examples:

```text
HTTP  → 80
HTTPS → 443
SSH   → 22
```

However, services can be configured to use alternative ports.

Example:

```text
HTTP → 8080
```

When a non-standard port is used, the port number must be specified in the URL:

```text
http://example.com:8080
```

---

## Key Notes

* Ports are logical communication endpoints.
* Port numbers range from 0–65535.
* Common ports exist between 0–1024.
* Ports help applications receive the correct network traffic.
* HTTP uses Port 80.
* HTTPS uses Port 443.
* SSH uses Port 22.
* FTP uses Port 21.
* SMB uses Port 445.
* RDP uses Port 3389.
* Services can run on custom ports when required.
