# Cloud Computing Fundamentals

## Conclusion

### Overview

Cloud Computing has become one of the most important technologies in modern IT. It enables organizations to access computing resources on demand without purchasing and maintaining expensive physical infrastructure. By combining virtualization, automation, and global data centers, cloud computing provides scalable, reliable, and cost-efficient services for businesses of all sizes.

This section summarizes the key concepts, service models, deployment types, and benefits learned throughout the room.

---

## Main Concept

### What is Cloud Computing?

Cloud Computing is the delivery of computing services over the internet, allowing users to access resources such as servers, storage, databases, networking, and applications whenever they need them.

Instead of owning hardware, organizations can rent resources from cloud providers and pay only for what they use.

---

## Important Components

### Public Cloud

Cloud infrastructure shared among multiple organizations and users.

Characteristics:

```text
Shared Infrastructure
Internet Accessible
Highly Scalable
Cost Effective
```

Examples:

```text
AWS
Microsoft Azure
Google Cloud Platform
```

---

### Private Cloud

Cloud infrastructure dedicated to a single organization.

Characteristics:

```text
Higher Security
Greater Control
Custom Configuration
Regulatory Compliance
```

Common Uses:

```text
Banking
Government
Healthcare
```

---

### Hybrid Cloud

Combination of Public Cloud and Private Cloud environments.

Characteristics:

```text
Flexible Deployment
Improved Security
Scalable Resources
Data Sharing Between Environments
```

Common Uses:

```text
Enterprise Applications
E-Commerce Platforms
Large Organizations
```

---

### Infrastructure as a Service (IaaS)

Provides virtual infrastructure resources.

User Manages:

```text
Operating System
Applications
Configurations
```

Provider Manages:

```text
Physical Hardware
Networking
Data Centers
```

Examples:

```text
AWS EC2
Azure Virtual Machines
Google Compute Engine
```

---

### Platform as a Service (PaaS)

Provides a managed environment for application development.

User Focuses On:

```text
Application Development
Code Deployment
Business Logic
```

Provider Manages:

```text
Infrastructure
Operating Systems
Runtime Environment
```

Examples:

```text
Google App Engine
Azure App Service
AWS Elastic Beanstalk
```

---

### Software as a Service (SaaS)

Provides fully managed software accessible over the internet.

Examples:

```text
Gmail
Zoom
Microsoft 365
Dropbox
```

Benefits:

```text
No Installation Required
Automatic Updates
Easy Accessibility
```

---

Common Uses:

## Advantages / Benefits

### Scalability

Resources can automatically increase or decrease according to demand.

---

### On-Demand Self-Service

Users can deploy and remove resources whenever required.

---

### Pay Only for What You Use

Organizations avoid large upfront infrastructure costs.

---

### Security

Cloud providers implement advanced security controls and monitoring systems.

---

### High Availability

Applications remain available even when hardware failures occur.

---

### Global Access

Services can be accessed from anywhere in the world through the internet.

---

## Key Characteristics

* Built on virtualization and containerization technologies.
* Resources are delivered through the internet.
* Supports rapid deployment and automation.
* Provides flexible scaling options.
* Uses a consumption-based pricing model.
* Enables global service delivery.
* Reduces infrastructure management complexity.
* Supports business continuity and disaster recovery.
